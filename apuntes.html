Curso de Introduccion a la Terminal y Linea de Comandos

Clase 1 - ¿Que es la terminal?

La terminal es un programa que ejecuta líneas de comandos, las cuales a su vez realizan acciones en el sistema operativo. Hay dos conceptos clave: la terminal y la shell. La terminal es una interfaz que recibe comandos y los pasa a la shell para su ejecución. La shell es la que realmente lleva a cabo el proceso de ejecutar los comandos, comunicándose con el procesador y la memoria RAM.

A diferencia de la interfaz gráfica convencional, la terminal permite interactuar con el sistema operativo mediante comandos. La interfaz gráfica es un programa adicional que ocupa espacio en la memoria, mientras que la terminal es más ligera y permite comunicarse directamente con los recursos del sistema operativo. Además, en situaciones donde la interfaz gráfica no está disponible o se daña, la terminal sigue siendo una herramienta útil.

Se recomienda utilizar la terminal en sistemas operativos basados en Linux o tener instalado el Windows Subsystem for Linux en Windows para seguir el curso.

Clase 2 - Instalar WSL - usa Linux dentro de Windows

En esta clase aprenderás a instalar el WSL (Windows Subsystem for Linux), una herramienta esencial para desarrolladores de software que utilizan Windows 10 o Windows 11. El WSL permite ejecutar una línea de comandos de Linux desde Windows, facilitando el desarrollo en un entorno Linux.

Si ya tienes una terminal basada en Unix instalada o utilizas Linux nativamente o macOS, puedes omitir esta clase. Sin embargo, si estás en Windows 10 u 11 y aún no tienes instalado WSL con algún sistema Linux, sigue las siguientes instrucciones detalladas:

1. Abre Windows PowerShell como administrador desde el menú de inicio.
2. Ejecuta el comando "wsl --install" para instalar el WSL. Este proceso permitirá ejecutar distribuciones de Linux como Ubuntu o Debian en tu computadora.
3. Espera a que la instalación llegue al 100% y presiona Enter nuevamente para iniciar la instalación del sistema operativo Ubuntu.
4. Reinicia tu computadora después de completar la instalación.
5. Configura Ubuntu proporcionando un nombre de usuario y contraseña.
6. Una vez reiniciada la computadora, accede a Ubuntu desde la aplicación Terminal en Windows.
7. Listo, ahora tienes WSL y Ubuntu instalados en tu máquina, lo que te permite utilizar una terminal de Linux directamente desde Windows para tus tareas de desarrollo.

Clase 3 - Aprendiendo a caminar en la terminal

En los sistemas operativos, la organización de archivos se realiza en una estructura de carpetas en forma de árbol jerárquico. En Linux, la carpeta raíz "/" contiene carpetas como "Home," que a su vez contiene carpetas para cada usuario.

Al abrir la terminal en Linux, se muestra información como el nombre de usuario, el nombre de la computadora, y la ubicación actual (~ indica la carpeta actual). El comando "pwd" muestra el directorio actual.

El comando "cd" (Change Directory) se utiliza para moverse entre carpetas en la terminal. Se pueden usar atajos como "~" para el directorio home, ".." para retroceder un nivel, y "/" para ir a la raíz.

El comando "ls" (List) muestra los archivos y carpetas en la ubicación actual. Con la opción "-l," se obtiene una lista detallada que incluye información como permisos, tamaño y fecha de modificación. La opción "-h" hace que los tamaños sean legibles para humanos.

El comando "file" proporciona información sobre el tipo de elemento seleccionado en una carpeta, como si es un directorio o un enlace simbólico.

Clase 4 - Manipulando archivos y directorios

Después de aprender a navegar y listar contenidos en la terminal, es crucial aprender a manipular archivos con comandos básicos en Linux.

1. Crear un directorio (mkdir):
- El comando "mkdir" (Make Directory) se utiliza para crear un directorio. Simplemente escribe el nombre del directorio que deseas crear, y puedes crear varios directorios al mismo tiempo.

mkdir DirectorioInteresante

2. Crear archivos (touch):
- El comando "touch" se utiliza para crear archivos. Simplemente escribe el nombre de los archivos que deseas generar delante del comando.

touch Secreto1 Secreto2 SecretoSecretario

3. Copiar archivos (cp):
- El comando "cp" (Copy) se utiliza para copiar archivos. Especifica el nombre del archivo que deseas copiar y el nombre con el que quieres que se guarde la copia.

cp Secreto1 Secreto_bk

4. Mover un directorio (mv):
- El comando "mv" (Move) se utiliza para mover archivos o directorios. Especifica el nombre del archivo que deseas mover y la ubicación a la que se moverá.

mv SecretoQueSeHizoPublico ../SecretosPublicos/

5. Eliminar archivos (rm):
- El comando "rm" (Remove) se utiliza para eliminar archivos o carpetas. Simplemente escribe el nombre o la ruta del elemento que deseas eliminar.

rm SecretoQueNadieDebeVer

- Opciones útiles de "rm":
- `-i` (interactive): Pregunta si estás seguro de eliminar el archivo.
- `-r` (recursive): Elimina todo lo que está dentro de una carpeta.
- `-f` (force): Fuerza la eliminación sin preguntar.


rm -i SecretoConfidencial
rm -r CarpetaConArchivos
rm -f ArchivoEnUso

touch: crea un archivo, si no indicas la extension por defecto sera .txt

cp [archivo]
[ruta]: nos permite duplicar archivos, para duplicar directorios con su contenido añadir el modificador -r que indica recursividad

mv [archivo]
[ruta] : mover un archivo hacia un directorio. También se usa para renombrar archivos mv [archivo] [nuevo nombre] no olvidar la extension del archivo Para mover directorios añadir el modificador -r

Clase 5 - Explorando el contenido de nuestros archivos

1. Comandos Head y Tail:
- Head imprime las primeras 10 líneas de un archivo, y tail imprime las últimas 10 líneas.
- Sintaxis simple: `head archivo.txt` o `tail archivo.txt`.
- Opción -n para especificar el número de líneas, por ejemplo: `head proyecto.html -n 20`.

2. Comando Less:
- Permite ver todo el contenido de un archivo y navegar por él.
- Sintaxis: `less [nombreDelArchivoParaAbrir]`.
- Interfaz gráfica para inspeccionar el documento.
- Uso de "/" para buscar palabras dentro del documento.
- Para salir de la interfaz, presiona "q".

3. Comando xdg-open:
- Abre un archivo en el programa predeterminado para su extensión.
- Ejemplo: `xdg-open archivo.html`.
- Puede requerir instalación con `sudo apt install xdg-utils` si no está disponible.
- Se utiliza para abrir archivos y, al ejecutarse para un archivo `.html`, muestra una interfaz similar a less.

4. Abrir una Carpeta con el Comando Nautilus:
- El comando `nautilus` abre una carpeta utilizando el programa Nautilus.
- Sintaxis: `nautilus carpeta`.
- Similar a xdg-open pero específicamente para carpetas.

5. Manejo de Errores:
- Si xdg-open no está instalado, seguir las instrucciones del mensaje de error para instalarlo.
- Ejemplo de instalación: `sudo apt install xdg-utils`.
En resumen, estos comandos permiten inspeccionar rápidamente el contenido de archivos, visualizar todo el documento de manera interactiva, abrir archivos en programas predeterminados y explorar carpetas desde la terminal.

Clase 6 - ¿Qué es un comando?

Un comando es un mensaje enviado al ordenador que provoca una respuesta en este sistema y se comporta como una orden, pues informa al dispositivo informático que debe ejecutar una acción según la indicación que pueda enviarse.

Un comando pueden significar cuatro cosas:

1. Un programa ejecutable
2. Un comando de utilidad de la shell. Esto es un programa en sí mismo, que puede tener funciones. Ejemplo cd
3. Una función de shell. Son funciones de shell externas al comando de utilidad. Ejemplo mkdir
4. Un alias. Un ejemplo es ls

Ejemplos de comandos básicos de la terminal
Ahora conocerás varios tipos de comandos que puedes aplicar en el proyecto que estás realizando.

- type [comando]: Nos permite conocer qué tipo de comando es 🤔.
- alias l="[secuencia de comandos]": Nos permite crear comandos. Son temporales, se borran al cerrar la terminal 👶🏼.
- help [comando]: Nos permite consultar un poco de documentación de un comando 📄.
- man [comando]: De manual, nos permite conocer mucha más información de un comando.
- info [comando]: Similar al anterior, pero un poco resumido y con otro formato.
- whatis [comando]: Describe un comando en una sola línea ☺️. No funciona con todos.

Clase 7 - Wildcards

Los wildcards o comodines son caracteres especiales que permiten realizar búsquedas más avanzadas en la manipulación de archivos y directorios. Se pueden utilizar con comandos como `ls`, `mv`, `cp`, y `rm`. Aquí hay un resumen de los tipos de wildcards mencionados:

1. Asterisco (*) - Buscar todo:
- `*`: Busca todos los archivos en una carpeta.
- `*.png`: Busca archivos con la extensión ".png".
- `fotoDe*`: Busca archivos que comiencen por "fotoDe".

2. Signo de interrogación (?) - Buscar por cantidad de caracteres:
- `?`: Busca un solo carácter.
- `foto?.png`: Busca archivos que sigan el patrón "fotoX.png" (donde X es un solo carácter).
- `foto??.jpg`: Busca archivos que sigan el patrón "fotoXX.jpg" (donde XX son dos caracteres).

3.Corchetes [] - Buscar por caracteres específicos:
- `[ci]`: Busca archivos que comiencen por "c" o "i".
- `[cCiI]*`: Busca archivos que comiencen por "c", "C", "i" o "I".
- `foto[2-6]*`: Busca archivos que comiencen por "foto" seguido de un número entre 2 y 6.

Estos wildcards ofrecen flexibilidad para realizar búsquedas más precisas y son útiles en diversas situaciones al manipular archivos en la línea de comandos.

Clase 8 - Redirecciones: cómo funciona la shell

El aprendizaje aborda el concepto de comandos y se centra en el manejo de entradas y salidas en la terminal mediante redirecciones. Se destacan las definiciones de Standard Input (stdin) y Standard Output (stdout), así como la introducción de file descriptors, especialmente el file descriptor 2 que representa Standard Error.

Se explican los operadores de redirección como > para enviar la salida estándar a un archivo y >> para concatenar la salida al final de un archivo existente. Además, se presenta la redirección de errores con la sintaxis 2>|2>&1, permitiendo dirigir el Standard Error hacia un archivo.

Se ilustran ejemplos prácticos, como guardar la salida de un comando en un archivo (ls -l > output.txt), concatenar información (ls -l >> output.txt image.png), y redirigir errores (ls -l > output.txt 2>&1). Se destaca la importancia de entender y controlar las entradas y salidas para optimizar el manejo de datos en la línea de comandos.

Operador Función
> Redirecciona la salida. Por defecto redirecciona el Standar Output
>> Concatena la salida con lo que ya tenga el archivo a donde se está redirigiendo la salida
2> Redirecciona el file descriptor 2 (En este caso Standar Error)
2>&1 Redirecciona el file descriptor 2 y 1

Clase 9 - Redirecciones: pipe operator

Pipe Operator:
- El Pipe Operator (`|`) permite tomar la salida de un comando y pasarla como entrada a otro comando.

Comandos de Pipe Operator:
1. Unir cadenas de texto (cat):

- `cat` se utiliza para concatenar la salida de varios comandos. Por ejemplo, para crear una lista de archivos de las carpetas "Images" y "SecretosDeEstado":

cat Images/* SecretosDeEstado/*

2. Crear un archivo con base en una salida (tee):
- `tee` se emplea para guardar la salida de un comando en un archivo. Ejemplo: cat images.txt secretosDeEstado.txt | tee archivos.txt

3. Organizar archivos con sort:
- `sort` ordena alfabéticamente la salida de un comando. Ejemplo para organizar la lista de archivos del directorio actual:

ls | sort | tee archivosHome.txt

- En este caso, se listan los archivos, se organizan alfabéticamente, y se crea un archivo llamado `archivosHome.txt` con esas salidas.

Estos comandos permiten realizar operaciones útiles en la línea de comandos, como unir información, guardarla en archivos, y organizarla de manera eficiente.

Comando Función
sort Organiza alfabéticamente una salida
cat Concatena dos entradas
tee Crea un archivo en base a una entrada

Clase 10 - Encadenando comandos: operadores de control

Operadores de Control en la Terminal

Los operadores de control son símbolos reservados que permiten encadenar comandos en la terminal. A continuación, se describen algunos de estos operadores:

1. Punto y coma (;):
- Permite ejecutar varios comandos en la misma línea, separándolos por el punto y coma.
- Ejemplo: `mkdir ProyectosSecretos; ls; date`

2. Ampersand (&):
- Facilita la ejecución eficiente de varios comandos simultáneamente.
- Ejemplo: `date & echo "Hola" & cal`

3. Condicionales AND (&&):
- Permite ejecutar comandos dependiendo de condiciones, asegurándose de que el comando anterior se ejecute correctamente
- Ejemplo: `cd lp && mkdir Comida`

4. Condicionales OR (||):
- Ejecuta comandos sin importar si el comando anterior tuvo éxito o no.
- Ejemplo: `cd ProyectosSecretos/ || cambia-carpeta ProyectosSecretos/`

5. Combinación de operadores:
- Se pueden combinar operadores para realizar acciones más complejas.
- Ejemplo: `cd ProyectosSecretos/ || cambia-carpeta ProyectosSecretos/ && mkdir ProyectoIncreible`

Estos operadores ofrecen flexibilidad y control en la ejecución de comandos, permitiendo la construcción de flujos condicionales y la ejecución concurrente de tareas en la terminal.

Operador Función
; Ejecuta de forma síncrona los comandos especificados
& Ejecuta de forma asíncrona los comandos especificados
&& Ejecuta el comando si el anterior se ejecutó correctamente
\|\| Ejecuta el comando si el anterior o la combinación de los anteriores resultaron en verdadero
Comando Función
echo Imprime el mensaje que le indiques
cal Imprime un calendario con la fecha actual
date Imprime la fecha actual

Clase 11 - Cómo se manejan los permisos

Los permisos en un sistema operativo determinan las acciones que cada usuario puede realizar sobre archivos y carpetas. Al listar archivos con "ls -l", la primera columna muestra los permisos. Los tipos de archivos incluyen normales ("-"), directorios ("d"), enlaces simbólicos ("l"), y archivos de bloque ("b").

Los permisos de usuario se dividen en Owner (dueño), Group (grupo), y World (otros). Se representan con los símbolos "r" (readable), "w" (writable), y "x" (executable). Los permisos se expresan en orden rwx, y un guion indica que el permiso no está disponible.

Ejercicio 1:
Permiso: drwxr-xr-x
Significado:
- Tipo: Directorio
- Dueño: Lectura, escritura y ejecución
- Grupo: Lectura y ejecución
- Otros: Lectura y ejecución

Ejercicio 2:
Permiso: -rw-r--r--
Significado:
- Tipo: Archivo normal
- Dueño: Lectura y escritura
- Grupo: Lectura
- Otros: Lectura

La representación numérica de permisos se organiza como rwx y se traduce a binario. Por ejemplo, r-x se convierte en 5. Usando este método, rwxr-xr-- se traduce a 751.

Ejercicios prácticos:
1. Directorio: 711 (rwx--x--x)
2. Enlace simbólico: 444 (r--r--r--)
3. Archivo común: 766 (rwxrw-rw-)

Practicar en la terminal o buscar información adicional puede ayudar a entender mejor este concepto.

Clase 13 - Cómo configurar variables de entorno

Las variables de entorno son útiles para priorizar cierta información o almacenar datos que no queremos recordar constantemente. En Linux, existen varias variables de entorno preestablecidas, como HOME, PATH, BASH_VERSION y SHELL, que se pueden ver desde la consola mediante comandos.

Para crear variables de entorno personalizadas, se puede acceder al archivo oculto ".bashrc" en el directorio de usuario. Se puede abrir con el comando "code .bashrc" (en VS Code) o editarlo directamente. Es importante tener precaución al modificar este archivo para evitar dañar la shell.

Se sugiere crear variables de entorno en mayúsculas por convención. Se puede ejemplificar creando una variable que almacene la ruta de una carpeta en Windows en el contexto de WSL. Después de guardar el archivo, reiniciar la terminal y ejecutar el comando, la variable de entorno se utiliza para cambiar al directorio deseado.

Además, se muestra cómo crear un alias que persiste incluso después de cerrar la terminal. Esto puede mejorar la eficiencia al crear alias para tareas frecuentes, como abrir VS Code en una carpeta específica. En resumen, la personalización de variables de entorno y la creación de alias pueden agilizar y facilitar el trabajo en el entorno de Linux.

Clase 14 - Comandos de búsqueda

A veces es necesario localizar y eliminar archivos innecesarios de un mismo tipo que ocupan espacio en el disco duro. Los comandos de búsqueda en la terminal son útiles para esta tarea, y uno de ellos es el comando `find`. Su sintaxis básica es `find [ruta] [opciones]`, y se pueden utilizar diversas opciones para segmentar la búsqueda.

1. Segmentar por nombre (-name): - Utilizando `-name`, se puede buscar archivos por su nombre o extensión en una ruta específica. Por ejemplo:
find ./ -name *.png

2. Segmentar por tipo (-type): - Con la opción `-type`, se pueden buscar archivos (-type f), directorios (-type d), o enlaces simbólicos (-type l). Se pueden combinar múltiples opciones separadas por comas. Ejemplo:
find ./ -type f -name "f*"

3. Segmentar por tamaño (-size):

- Se puede utilizar la opción `-size` para buscar archivos según su tamaño. Se especifica la unidad de medida (c, k, M, G) y se puede usar el símbolo + para mayor o - para menor. Ejemplos:

find ./ -size 4k
find ./ -size +4k
find ./ -size -4k

4. Buscar archivos vacíos (-empty):

- La opción `-empty` se usa para buscar archivos o directorios vacíos. Ejemplo:
find ./ -type d -empty

5. Limitar la búsqueda (-maxdepth, -mindepth):

- Se pueden limitar las carpetas a explorar con `-maxdepth` (profundidad máxima) y `-mindepth` (profundidad mínima). Ejemplos:
find ./ -type d -maxdepth 2
find ./ -type d -mindepth 2

6. Uso de `less` para mejorar la visualización:

- Se recomienda pasar la salida del comando `find` a `less` para facilitar la exploración de resultados:
find ./ | less

Estos comandos permiten una búsqueda efectiva y personalizable en la terminal para gestionar y eliminar archivos de manera eficiente.

Clase 15 - Usando el comando grep

Uso del comando `grep` para buscar texto dentro de archivos:

Significado de "grep":
- "Grep" proviene de "Global Regular Expression Print" y utiliza expresiones regulares (regex) para realizar búsquedas dentro de archivos.

Sintaxis básica:
- La sintaxis general es: `grep [ExpresiónRegular] [archivoDondeBuscar]`.

Ejemplo práctico:
- Si tenemos un archivo llamado "movies.csv", podemos buscar palabras dentro de él con el comando: `grep the movies.csv`.

Opciones útiles:

1. Ignorar case sensitive (-i):
- La opción `-i` permite buscar sin distinguir entre mayúsculas y minúsculas. Ejemplo:

grep -i Action movies.csv

2. Contar ocurrencias (-c):
- Utilizando `-c`, se puede obtener el conteo de cuántas veces aparece una palabra en el archivo. Ejemplo:

grep -c Drama movies.csv

3. Excluir una expresión (-v):
- La opción `-v` permite mostrar resultados que NO coincidan con la expresión regular proporcionada. Ejemplo:

grep -cv Drama movies.csv

4. Limitar la búsqueda (-m):
- Con `-m` se limita la búsqueda a un número específico de líneas. Ejemplo:

grep -m 10 Fan movies.csv

Estas opciones amplían la versatilidad del comando `grep`, facilitando la búsqueda y análisis de información dentro de archivos de texto.

Clase 16 - Utilidades de red

Se aborda el manejo básico de redes informáticas, centrándose en comandos esenciales para obtener información y realizar algunas acciones comunes:

Configuración de Dispositivos (ifconfig):

Se utiliza el comando ifconfig en la consola para obtener información sobre el dispositivo de red (por ejemplo, "eth0"), su configuración, dirección IPv4 e IPv6, y máscara de red. El comando netstat también se menciona como una opción para visualizar la información de red de manera más amigable.

Se utiliza el comando ping seguido de la URL para verificar la disponibilidad de una página desde la dirección IP.

Obtención de Archivos de una Página (curl | wget):

Se emplea el comando curl para obtener información de una página web y mostrarla en la consola.
El comando wget realiza una acción similar, pero guarda la información en un archivo especificado.
Se menciona la posibilidad de descargar múltiples páginas al mismo tiempo mediante wget.

Ruta de Acceso a la Página (traceroute):

Se introduce el comando traceroute para visualizar la ruta que sigue una conexión a una página en Internet.
Se explica que la conexión no se realiza directamente al servidor de la página, sino a través de intermediarios (servidores) en el camino.

Este resumen proporciona una visión general de cómo utilizar estos comandos para obtener información sobre la red, verificar la disponibilidad de páginas web, obtener archivos de sitios y visualizar la ruta de acceso a una página a través de la red.

Clase 17 - Comprimiendo archivos tar y zip

Compresión con formato .tar:


1. Utilizamos el comando `tar` para comprimir archivos en formato `.tar`.
2. La sintaxis básica es `tar -cf [nombreDelArchivoComprimido] [archivoAComprimir]`.
3. Para visualizar el progreso de la compresión, se agrega la opción `-v` o `--verbose`.
- Ejemplo: `tar -cvf compressed.tar Documents/toCompress/`.
4. Para comprimir en formato `.tar.gz` o `.tgz`, se añade la opción `-z`.
- Ejemplo: `tar -czvf compressed.tar.gz Documents/toCompress/`.
5. La descompresión se realiza con la opción `-x`.
- Ejemplo: `tar -xzvf compressed.tar.gz`.

Compresión con formato .zip:

1. Utilizamos el comando `zip` para comprimir archivos en formato `.zip`.
2. La sintaxis básica es `zip -r [nombreDelArchivoComprimido] [archivoAComprimir]`.
- La opción `-r` es utilizada para comprimir carpetas de forma recursiva.
- Ejemplo: `zip -r compressed.zip Documents/toCompress/`.
3. La descompresión se realiza con el comando `unzip`.
- Ejemplo: `unzip compressed.zip`.

Estos comandos son útiles para comprimir y descomprimir archivos en sistemas operativos basados en UNIX, como Linux. Se pueden emplear para organizar y reducir el tamaño de archivos y carpetas.

Opciones del Comando tar:

Opción Función
c Comprimir
x Descomprimir
